  - Página `/dashboard/brands` completa: grid, skeleton de loading, empty state

## Decisões técnicas

- **Tipos gerados pelo Supabase MCP**: A tipagem manual era incompatível com `postgrest-js v2.x` (novo formato `__InternalSupabase` com `PostgrestVersion`). Gerado via MCP e substituído completamente.
- **Slug automático**: Gerado no cliente a partir do nome (normalização NFD + regex) — apenas na criação; edição preserva o slug existente.
- **Unique constraint no banco**: `unique(user_id, slug)` na tabela `brands` — tratado no frontend com mensagem específica no erro `23505`.
- **Storage bucket `logos`**: público para leitura, write autenticado — logos são URLs públicas permanentes.

## Problemas resolvidos

- **`type never` no Supabase**: postgrest-js v2 exige novo formato de tipo com `__InternalSupabase.PostgrestVersion` — resolvido gerando os tipos reais.
- **ThemeProvider ausente**: o `sonner.tsx` do shadcn usa `useTheme` do next-themes — adicionado ThemeProvider no layout.
- **Tailwind não aplicava no próximo componente**: não foi um problema nesta sessão.
