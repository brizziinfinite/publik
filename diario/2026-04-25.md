# Diário — 25 de abril de 2026

## O que foi feito

- Bucket `media` criado no Supabase Storage (50MB, imagens + vídeos) com policies RLS
- Store Zustand atualizado: campo `brand` → `activeBrand` com persist no localStorage (só persiste a brand, não o user)
- Seletor de brand na Sidebar: dropdown com todas as brands do usuário, auto-seleção da primeira, indicador da ativa com `✓`, atalho "Nova brand"
- CRUD completo de Posts:
  - `usePosts` hook reativo filtrado por `brand_id`
  - `PostDialog` — criação/edição com textarea, select de plataforma, select de status, date picker (calendar + popover), upload múltiplo de mídia
  - `PostCard` — grid com badge de status colorido, plataforma, preview do conteúdo (3 linhas), contagem de mídias, data agendada
  - `DeletePostDialog` — confirmação de exclusão
  - Página `/dashboard/posts`: filtros por status (pills), grid, skeleton, empty states (sem brand / sem posts / sem posts no filtro)

## Decisões técnicas

- **Zustand persist partializado**: só `activeBrand` vai para localStorage, `user` é reconstituído pela sessão Supabase a cada carregamento — evita stale data de auth.
- **Gate de brand na página Posts**: se não há brand ativa, exibe CTA direto para `/dashboard/brands` em vez de quebrar.
- **Zod v4**: `required_error` não existe mais — removido, usando apenas mensagem padrão do enum.
- **Mídia existente vs nova**: ao editar, mídias existentes são exibidas por URL; novas são blob URL; remoção trata cada caso separado.

## Problemas resolvidos

- **Zod v4 `required_error`**: API mudou — removido o campo, build passou.
