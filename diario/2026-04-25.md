## Decisões técnicas

- **Calendário customizado**: não usamos FullCalendar nem o Calendar do shadcn (date picker). Grid CSS 7 colunas com date-fns — sem dependências extras.
- **SidebarContent extraído**: componente único reutilizado pelo Sidebar desktop e pelo Sheet mobile. Aceita `onNavClick` para fechar o Sheet ao navegar.
- **Avatar no bucket `logos`**: pasta `avatars/{user_id}.{ext}` dentro do bucket existente — sem criar novo bucket.
- **Stats via hook**: `usePosts` já carrega todos os posts — contadores calculados com `useMemo`, sem queries extras ao banco.
- **Preview sem dependências externas**: layouts feitos com Tailwind puro, sem biblioteca de mock.
- **Reset password flow**: `resetPasswordForEmail` envia email com link para `/reset-password`; o Supabase injeta sessão temporária via hash na URL, então `supabase.auth.updateUser({ password })` funciona diretamente.
- **`suppressHydrationWarning` no `<html>`**: necessário para o next-themes injetar a classe `dark`/`light` via script inline antes do hydration.

## Problemas resolvidos

- **`type never` em Supabase insert/update**: tipos gerados manualmente incompatíveis com postgrest-js v2. Resolvido gerando types via MCP e incluindo `__InternalSupabase: { PostgrestVersion: "14.5" }`.
- **Zod v4 `required_error`**: API mudou — removido o campo dos validators de enum.
- **`post.content` possivelmente null**: TypeScript error corrigido com `(post.content ?? "")` nas páginas de calendário e dashboard.
