## Decisões técnicas

- **Route group mantido**: as páginas do pacote foram colocadas em `app/(dashboard)/dashboard/sources`, preservando a arquitetura atual.
- **Fila lazy no Next**: `lib/queue.ts` só cria conexão Redis quando a API é chamada, evitando quebra durante build.
- **Worker separado**: o worker ficou fora do build do Next e foi excluído do `tsconfig.json` principal.
- **Tabs local**: criado componente simples compatível com o uso do pacote, sem adicionar outra dependência UI.
- **Env compartilhado**: o worker lê `.env.local` da raiz para facilitar execução local.

## Problemas resolvidos

- **Pacote não era app executável**: identificado como pacote de implementação para ser colado no projeto principal.
- **Imports Supabase incompatíveis**: pacote esperava `createServerClient`/`createBrowserClient`; adaptado para `createClient`.
- **Worker com NodeNext exigindo `.js` nos imports**: `tsconfig` do worker ajustado para `moduleResolution: Bundler`.
- **Patch separado de regeneração**: consolidado no arquivo principal da fila.
- **Build do Next com worker incluído**: `worker` excluído do `tsconfig.json` principal.
