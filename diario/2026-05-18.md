## Sessão 2 — n8n: Workflow 📬 e-mail Gestor

### O que foi feito

- **Diagnóstico e fix do Merge node**: `numberInputs` estava em 5, mas o fluxo Gmail tem 4 contas → Merge travava esperando 5° input. Corrigido para `numberInputs: 4`
- **Fix conexões por ID vs nome**: edições via API estavam usando IDs nos `connections`, mas n8n espera nomes de nó. Corrigido para usar nomes
- **Fix campos Gmail capitalizados**: Gmail node retorna `Subject`, `From`, `To` (maiúsculo) mas o código usava `subject`, `from`, `to`. Corrigido com `email.Subject || email.subject`
- **Fix Categorizar**: código esperava `$input.first().json.emails` (array dentro de objeto), mas Filtrar Importantes retorna N items individuais. Corrigido para `$input.all().map(item => item.json)`
- **Refino de filtros**: lógica `!isSpamKw && !isSpamDomain` passava 192 emails. Substituído por blocklist de domínios + keywords financeiras/operacionais estritas → 3-5 emails relevantes por ciclo
- **Deduplicação**: adicionado `seen = new Set()` por `id` em Filtrar Importantes e Filtrar Spam
- **Keywords refinadas**: `das ` → `guia das`, `vence` → `vence hoje`, adicionado `falha ao realizar o pagamento`, removido false positives
- **Fluxo de anexos**: adicionados 5 nós novos (Buscar Email Completo, Extrair Anexos, Tem Anexo?, Download Anexo, Evolution Enviar Anexo). Quando email importante tiver anexo PDF/XML → envia arquivo direto no WhatsApp

### Decisões técnicas

- **Blocklist de domínios > spamKeywords**: mais preciso. Substack, NoCode newsletters, BTG comunicados, etc. bloqueados por domínio
- **Gmail typeVersion 2 com `downloadAttachments: true`**: único modo que retorna binários dos anexos sem precisar de HTTP Request separado com OAuth
- **Fluxo de anexo paralelo ao texto**: Filtrar Importantes → Categorizar (texto) + Buscar Email Completo (anexo) em paralelo. Não bloqueia o resumo texto se não tiver anexo

### Problemas resolvidos

- Merge travado aguardando 5° input que nunca chegava (iCloud tem trigger separado)
- n8n `$output` API não existe em v2.16.1
- HTTP Request com `predefinedCredentialType: gmailOAuth2` não funciona (credencial não exposta)
- Gmail node `get` com `format: full` no parâmetro era ignorado (node retorna formato simplificado sempre)
- Conexões via API criavam usando IDs → visual quebrado no editor e execução falha
