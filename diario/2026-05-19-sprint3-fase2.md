## Decisões técnicas

- `visual_identity_v2` é **cópia** dos tokens do kit (não FK) — brand pode customizar depois sem afetar o kit base
- `as unknown as VisualKit[]` necessário porque Supabase tipifica campos jsonb como `Json` (opaco) — cast duplo é padrão aceito para dados jsonb tipados via interface manual
- Migration idempotente (ON CONFLICT DO UPDATE) — pode rodar várias vezes sem duplicar
