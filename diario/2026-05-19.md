- Retry apenas em 503/429/529 e erros de rede — outros erros falham direto

- `supabase db push` executado — migrations Sprint 1 + Sprint 2 aplicadas no remoto
- `agent-2-roteirista` deployado com sucesso

### Decisões técnicas

- Helper em `_shared/` para reutilizar sem duplicação
- `withRetry` nunca throwa — retorna `RetryResult<T>`, caller decide
- `input_payload` enriquecido com `attempts` via update após chamada LLM

### Problemas resolvidos

- **Gemini 503 em produção**: antes marcava run como `failed`. Agora retries com backoff
- **ANTHROPIC_API_KEY vazia**: `.env.local` não tinha a key — Gemini voltou antes de precisar do fallback

### O que foi feito

- `lib/render/fonts.ts`: loader com cache `Map<string, ArrayBuffer>`

- `lib/supabase/service.ts`: service role client
- `app/api/render-package/route.ts`: POST, `runtime = 'nodejs'`, idempotente com flag `re_render`
- Botão "Renderizar carrossel" + galeria de PNGs com download individual em `/dashboard/packages/[id]`
- `types/database.ts` regenerado com `render_formats` + alias `RenderFormatRow`
- `types/render-format.ts`: interfaces `RenderFormat`, `RenderRequest`, `RenderResponse`, `RenderedSlide`
- Teste ao vivo: carrossel IrrigaAgro → 7 PNGs em 7s, URLs no Storage ✅

### Decisões técnicas

- `runtime = 'nodejs'` na API Route — Edge runtime não tem `fs/promises`
- FK hint explícito: `content_ideas!content_packages_idea_id_fkey` e `brands!content_packages_brand_id_fkey` — múltiplas FKs causavam ambiguidade no Supabase JS
- `safe_zone as unknown as RenderFormatSafeZone` — jsonb é `Json` opaco no Supabase
- `reloadRef` pattern — permite chamar `load()` de handler async sem violar `react-hooks/set-state-in-effect`
- Template usa apenas `display: flex` + inline styles — restrição do Satori (sem grid, sem Tailwind)
- Projeto usa npm + `package-lock.json` — **não usar pnpm**, cria conflito de lock files

### Problemas resolvidos

- **React duplicado / ThemeProvider crash**: `@vercel/og` instalado com pnpm criou `pnpm-lock.yaml` conflitando com `package-lock.json`. Turbopack instanciava React duas vezes. Solução: remover pnpm artifacts, reinstalar com npm + limpar `.next`
- **"More than one relationship found"**: Supabase JS ambíguo na FK entre `content_packages` e `content_ideas`. Solução: nome explícito da FK no select
- **`render_formats` fora dos tipos**: `database.ts` desatualizado. Regenerado com `SUPABASE_ACCESS_TOKEN=... supabase gen types typescript`
- **`safe_zone as RenderFormatSafeZone` falhou no build**: jsonb → `as unknown as` necessário
