## Decisões técnicas

- `convertToPost.ts` usa `TablesInsert<"posts">` em vez de `as never` para type safety
- Tabela `content_packages` antiga no banco tinha schema diferente — foi dropada e recriada (zero rows)
- Índice parcial `WHERE status not in (...)` requer DDL em migration separada ou `DROP + CREATE` (bug do Supabase MCP: erro "column status does not exist" quando existe outra tabela com mesmo nome e schema diferente)
- `eslint-disable-line react-hooks/set-state-in-effect` nos `useEffect(() => { void loadX(); })` — padrão existente no projeto

## Problemas resolvidos

- **`content_packages` antiga**: `CREATE TABLE IF NOT EXISTS` não criou nova estrutura. Solução: `DROP TABLE cascade` + `CREATE TABLE` fresh
- **`convertToPost.ts` build error**: `.from("posts" as never).insert(...)` quebrava tipo → reescrito com `TablesInsert<"posts">`
- **Deploy 401**: CLI Supabase sem sessão. Requer `supabase login` interativo pelo usuário
