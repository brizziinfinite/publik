## Decisões técnicas

- `convertToPost.ts` usa `TablesInsert<"posts">` em vez de `as never` para type safety
- Tabela `content_packages` antiga no banco tinha schema diferente — foi dropada e recriada (zero rows). Nunca usar `CREATE TABLE IF NOT EXISTS` ao mudar schema sem checar se a tabela existente é compatível
- Índice parcial `WHERE status not in (...)` falha com erro "column status does not exist" quando a tabela já existe com nome igual mas colunas diferentes — resolver com DROP + CREATE fresh
- `eslint-disable-line react-hooks/set-state-in-effect` nos `useEffect(() => { void loadX(); })` — padrão existente no projeto, não alterar
- `key={idea.id}` no EditDialogInner para reset de estado sem useEffect — padrão a manter em dialogs que recebem props mutáveis

## Problemas resolvidos

- **`content_packages` antiga com schema incompatível**: `CREATE TABLE IF NOT EXISTS` não recriou. Solução: checar com `SELECT column_name FROM information_schema.columns` antes de aplicar migration
- **`convertToPost.ts` build error**: `.from("posts" as never).insert(...)` quebrava tipo → reescrito com `TablesInsert<"posts">`
- **Deploy 401**: CLI Supabase sem sessão ativa — usar token de acesso via `SUPABASE_ACCESS_TOKEN=<token> supabase functions deploy ...`
- **ReelContent sem cenas**: prompt gera `scenes[]` mas componente só renderizava `hook_3s` e `cta_final` — corrigido com tipo e render completo

### Decisões técnicas

- `runtime = 'nodejs'` na API Route — Edge runtime não tem `fs/promises` para carregar fontes
- FK hint explícito no select: `content_ideas!content_packages_idea_id_fkey` — múltiplas FKs entre `content_packages` e `content_ideas` causavam erro de ambiguidade no Supabase JS
- `safe_zone as unknown as RenderFormatSafeZone` — jsonb do Supabase tipado como `Json` opaco, cast duplo necessário
- `reloadRef` pattern no componente React — permite chamar `load()` de dentro de handler async sem violar regra `react-hooks/set-state-in-effect`
- Template usa apenas `display: flex` e inline styles — restrição do Satori (sem grid, sem classes CSS)
- Projeto usa npm + `package-lock.json` — não usar pnpm, cria conflito de lock files e React duplicado

### Problemas resolvidos

- **React duplicado / ThemeProvider crash**: instalar `@vercel/og` com pnpm criou `pnpm-lock.yaml` conflitando com `package-lock.json`. Turbopack não resolvia React corretamente. Solução: remover `pnpm-lock.yaml` + `pnpm-workspace.yaml`, reinstalar com npm
- **"More than one relationship found"**: Supabase JS não sabia qual FK usar entre `content_packages` e `content_ideas`. Solução: `content_ideas!content_packages_idea_id_fkey(...)` e `brands!content_packages_brand_id_fkey(...)`
- **`render_formats` fora dos tipos**: `types/database.ts` desatualizado — regenerado via `SUPABASE_ACCESS_TOKEN=... supabase gen types typescript`
- **`safe_zone as RenderFormatSafeZone` falhou no build**: jsonb retorna tipo `Json` — corrigido com `as unknown as RenderFormatSafeZone`
