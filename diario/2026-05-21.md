## Decisões técnicas

- **Upload de logo no handleNext, não no onChange**: evita upload desnecessário se usuário cancela ou troca de arquivo várias vezes
- **getMissingFields centralizado no page**: steps são "burros" — recebem `missingFields` como prop, não calculam. Elimina divergência entre lógica de validação e feedback visual
- **canAdvance = missingFields.length === 0**: uma linha, sem duplicação
- **Visual kit: `id` é string (não UUID)**: ex: `"agro_serio"`, `"boutique_feminina"` — importante para queries
- **Formatos checkboxes com rank**: numeração mostra prioridade por ordem de seleção, sem precisar de drag & drop

## Problemas resolvidos

- **content_formats_priority era string (textarea)**: migrado para string[] (checkboxes). Requer atualizar 3 lugares: WizardState, brandToState, buildPayload
- **Build falha sem .env.local**: erro pré-existente — Next.js tenta prerender `/dashboard/ideas` sem variáveis Supabase. Não é regressão. Build passa na Vercel (tem env vars)
